# レイアウトプロトコル詳細解析結果

## 解析対象データ
- `captures/layout_2x4_serpentine.txt` — 2列×4行 S字パターン (56コマンド)
- `analysis/layout_preset_rightToLeft.txt` — 4列×1行 R→L / L→R (各42コマンド)

## コマンド構造比較

| カテゴリ | 2×4 (56cmd) | 4×1 (42cmd) | 備考 |
|---------|-------------|-------------|------|
| 受信カード初期化 (dest=0xFF) | 4 | 4 | 固定 |
| グローバル設定 (画面サイズ等) | 11 | 11 | 画面サイズのみ可変 |
| その他グローバル | 2 | 2 | 固定 |
| マッピングテーブル (256Bブロック) | 16 | 16 | データ内容が可変 |
| パーカード設定 | 16 | 8 | カード数に比例 (cards × 2) |
| カードグローバル (board=ALL) | 1 | 1 | 固定 |
| ステータスリード | 6 | 0 | NovaLCTの確認動作、送信不要 |

## 画面サイズレジスタ

キャビネットサイズ: 128×128 px (固定)

| レジスタ | 名称 | 2×4 (2col×4row) | 4×1 (4col×1row) | 計算式 |
|---------|------|-----------------|-----------------|--------|
| 0x02000024 | area1_width | 256 (2×128) | 512 (4×128) | columns × cabinetWidth |
| 0x02000026 | area1_height | 512 (4×128) | 128 (1×128) | rows × cabinetHeight |
| 0x0200002C | area1_stride | 256 | 512 | = width |
| 0x02000051 | area3_width | 256 | 512 | = area1_width (ミラー) |
| 0x02000053 | area3_height | 512 | 128 | = area1_height (ミラー) |
| 0x02000028/55 | x_offset | 0 | 0 | 常に0 |
| 0x0200002A/57 | y_offset | 0 | 0 | 常に0 |

## カード数レジスタ (0x03100000)

- 2×4: **8** (= 2 × 4)
- 4×1: **4** (= 4 × 1)
- **計算式**: `columns × rows`

## マッピングテーブル構造

- 16ブロック × 256バイト、**全ブロック同一データ**
- 各エントリ: **4バイト** `[X_LE16][Y_LE16]` — カード1枚分の画面上座標
- エントリ数 × 4バイトのパターンが256バイトになるまで繰り返し

### マッピングテーブル例

**4×1 L→R** (4エントリ × 4B = 16B、16回繰り返し):
```
(0,0), (128,0), (256,0), (384,0)  ← X昇順、Y=0
```

**4×1 R→L** (4エントリ × 4B = 16B、16回繰り返し):
```
(384,0), (256,0), (128,0), (0,0)  ← X降順、Y=0
```

**2×4 S字** (8エントリ × 4B = 32B、8回繰り返し):
```
(0,384), (0,256), (0,128), (0,0),      ← col0: 下→上
(128,0), (128,128), (128,256), (128,384) ← col1: 上→下
```

### マッピングテーブルの意味

エントリ[i] = ボード[i] の画面上表示座標 (X, Y)。
スキャン方向はこの座標配置で決定される。

## パーカード設定

- ポート: **0x00** (全パターン共通)
- ボードインデックス: 0, 1, 2, ... (カード番号)
- 送信順序: マッピングテーブルのエントリ順と一致
- レジスタ:
  - `0x02000017` (X_size) = cabinetWidth (常に128)
  - `0x02000019` (Y_size) = cabinetHeight (常に128)

### ボード送信順序

| パターン | 順序 | 意味 |
|---------|------|------|
| 4×1 L→R | [0, 1, 2, 3] | 左→右 |
| 4×1 R→L | [3, 2, 1, 0] | 右→左 |
| 2×4 S字 | [3, 2, 1, 0, 4, 5, 6, 7] | col0下→上、col1上→下 |

## コミットコマンド

- `reg=0x020000AE, data=0x01` (dest=0xFF) — レイアウト適用
- `reg=0x01000012, data=0xAA` (dest=0xFF, port=0x01, board=0xFFFF) — 設定保存
- `reg=0x020001EC, data=[width_LE16, height_LE16]` (dest=0x00) — 最終確定

## スキャン方向の実装方針

スキャン方向は **マッピングテーブルの座標配置** と **ボード送信順序** のみで決定。
専用の「スキャン方向レジスタ」への書き込みは不要。

### S字パターンのアルゴリズム

```
for col in 0..<columns:
    if col % 2 == 0:  # 偶数列: 下→上
        for row in (rows-1)...0:
            entry = (col * cabinetWidth, row * cabinetHeight)
    else:             # 奇数列: 上→下
        for row in 0..<rows:
            entry = (col * cabinetWidth, row * cabinetHeight)
```

## 既存コードのバグ

1. **カード数レジスタ**: `columns` を送信 → **`columns × rows`** に修正必要
2. **マッピングブロック**: 8バイト/エントリ → **4バイト/エントリ** に修正必要
3. **パーカードのポート**: `0x01` → **`0x00`** に修正必要
4. **scanDirection未使用**: `setLayout`にscanDirectionパラメータなし → 追加必要
